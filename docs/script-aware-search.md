# 简繁字形感知检索

## 范围

每次检索都在一个已完成或部分完成的任务 OCR 语料中执行，不重新 OCR，也不覆盖
原文。检索会话、查询形式、命中证据和统计结果写入任务 SQLite，因此同一任务可重复
搜索不同词语，并保留可审计历史。

范围取值如下：

| 范围 | 命中规则 |
| --- | --- |
| `simplified` | 原始图片字形为简体，或全部为简繁共用字 |
| `traditional` | 原始图片字形为繁体，或全部为简繁共用字 |
| `both` | 简体、繁体、共用字和简繁混合字形 |

范围判断以 `raw_text` 对应位置为准，不以 OpenCC 转换结果冒充原始字形。发生长度
变化而无法可靠映射回原文的位置只允许在 `both` 中出现。孤立单字的非主 Top-K
候选按候选字形判断范围，并明确标记为待人工复核；payload 同时保留主 OCR 原文，
不把候选写回语料。

## 证据层

同一 OCR 行同一原文位置最多保留一个命中，按以下顺序选择证据最强者：

1. `raw_exact`：OCR 上下文原文与输入完全一致；
2. `context_resolved`：同一模型的上下文识别结合高置信孤字结果后与输入一致；
3. `variant_graph`：简体、标准繁体、台湾或香港 OpenCC 索引图匹配；
4. `ocr_top_k`：孤立单字的非主 Top-K 候选，状态固定为
   `candidate_pending_review`。

结果保存命中层、索引种类、原文位置、原始字形类别、置信度和用于判定的 payload，
不把较弱候选伪装成精确命中。

## 一简多繁与词语语义

单字查询通过 OpenCC `t2s` 反向关系生成一简多繁候选图。例如“发”同时保留
“發”“髮”等候选。单字没有上下文时统一标记：

```text
仅字形关联，语义未确认
```

词语查询分别计算整词 OpenCC 转换和逐字转换。只有二者不同时，才记录对应地区的
`opencc_phrase_evidence` 并标记“OpenCC 词语映射支持”；否则仍使用字形图命中，
但不得声明已完成语义消歧。该规则不引入外部词典、网络服务或付费能力。

## 旧任务与不完整任务

- `ready`：完整语料，可检索；
- `partial`：允许检索，但会话统计必须带 `corpus_incomplete=true`；
- `legacy_requires_reocr`：拒绝生成推测性结果，要求用户明确创建重新 OCR/索引任务；
- `not_built`、`building`、`failed`：当前不可检索。

检索会话与全部命中在一个 SQLite 事务中提交；任何命中写入失败都会连同会话整体
rollback。
