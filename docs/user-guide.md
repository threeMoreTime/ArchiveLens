# 用户使用说明

## 创建检索任务

1. 打开“新建扫描”，选择整个本地文件夹、单个文件，或最多 200 个 PDF、DJVU、DJV、TIFF、JPEG、PNG 文件组成的清单。文件夹会先进行可取消预检，显示有效文件数、格式、总体积、已知页数、可用磁盘、保守空间估算和风险明细；文件夹超过 200 个文件不会被硬性拒绝，但大任务、网络路径、跳过的 junction/符号链接或磁盘余量偏低时必须明确确认。阻塞项修复后需要重新预检。多个显式文件可来自不同目录并可混合格式；重复文件会自动去重。
2. 输入 1～32 个 Unicode code point 的单个文字或连续词语。“设置 → 简繁字形检索”的当前默认范围会随任务固化：可只命中原图简体、只命中原图繁体，或两者都命中，默认两者都命中；OpenCC 仅用于索引和坐标映射，不转换覆盖 OCR 原文。
3. 如需处理古籍竖排，先在“设置 → 校对显示与扫描上下文”中选择阅读方向和关键词每侧 1～50 字的上下文长度，再点击“开始扫描”。出处页始终按源文件无损显示；上下文默认“从左到右、每侧 15 字”，任务页可使用原来源重新创建任务并继承该任务的单独设置。
4. 在任务页观察文件数、页进度、命中数和失败明细。总页数尚未统计完成时，界面显示不确定进度，不再显示误导性的 `N/0`。
   升级后恢复尚未完成的旧任务时，应用会从已经保存的 OCR 索引补齐已扫描页命中，不重新 OCR，也不改变暂停断点；回填结果仍需按原图人工核查。
5. 扫描完成后进入校对工作台。标题说明和横向摘要不占用工作区；除最右 56px 待处理进度窄栏外，页面按约 50% / 25% / 25% 分为原文件整页图、筛选与结果列表、完整上下文与校对操作。每条结果最左显示任务内永久序号（如 `#0001`），默认按序号升序，筛选、翻页、校对更新和后续扫描追加都不会改变旧序号。原文件图纵向占满左栏，支持滚轮缩放、拖拽、适应窗口、重新居中和朝上/右/下/左旋转，最大 400%；同一源文件的展示方向会保存并在重开应用后恢复，但不修改原文件、OCR 坐标、列表缩略图或导出内容。PDF 会按显示物理像素从原文件重新渲染，位图和 DjVu 超过 100% 时只放大观察并提示。完整上下文会自动换行，超长内容在右栏内滚动；1080px 最小窗口仍保持三栏，各区独立滚动且控件可换行。
6. 按“确认命中 / 需要复核 / 拒绝命中”完成判断，再使用“上一条 / 下一条待处理 / 下一条”继续浏览。备注停顿约 700ms 后自动保存；切换结果、分页或进入导出中心前也会提交当前草稿，`Ctrl+Enter` 可立即保存。数据库写入失败时，草稿会暂存于本机并阻止工作台内的切换操作，返回该结果后可继续重试。
7. 在导出中心选择 JSON 或离线 HTML，再点击一次主按钮创建后台导出作业。JSON 中每条命中包含同一个永久序号；HTML 中每条命中按“序号与出处 → OCR 文字和校对信息 → 完整原始页面”独立成卡。报告会逐页验证并嵌入源文件派生的无损 PNG，PDF 至少使用 384 DPI；同一页面即使有多条命中也只编码一次图片数据。导出中心会显示排队和阶段进度，支持取消，以及对失败、取消或应用中断的作业重新导出。任一页面无法验证或生成时，本次导出会失败且不会覆盖已有报告；导出前需要确认无损大图片报告可能超过 300MB。扫描或校对未完成时，确认区还会同时说明阶段性报告的完整性风险。
   HTML 报告的桌面命中导航默认展开，可用标题栏左箭头完全收起，再用页面左侧带命中数量的悬浮右箭头恢复；当前记录会随正文滚动高亮。窄窗口会改用正文上方的横向展开栏，打印时导航自动隐藏；每次重新打开报告仍从展开状态开始。

## 任务内简繁检索与人工核查

1. 新版本任务完成至少一页 OCR 持久化后，可从任务详情或左侧“任务内检索”进入。
2. 输入单字或词语，选择“只命中简体”“只命中繁体”或“简体和繁体”。全局默认在“设置 → 简繁字形检索”中修改，初始值为“简体和繁体”。
3. 结果按“原文精确命中、上下文识别命中、简繁字形索引命中、OCR Top-K 候选”分层显示。同一位置只保留证据最强的一条；Top-K 候选不是 OCR 主结果，必须人工核查。
4. 选择结果后，在完整出处页中缩放并查看高亮。详情同时显示不可变 OCR 原文、上下文识别文本、图片原字形、匹配索引和置信度；OpenCC 只生成检索索引，绝不覆盖原文。
5. 每次检索会话及其命中证据保存在当前任务 SQLite 中。可从历史列表重复打开，不需要重新 OCR。

`partial` 语料允许检索，但界面会提示扫描仍在进行或存在失败页，结果可能增加或漏检。
旧任务显示“需重新 OCR”，不会根据旧命中伪造全文，也不会静默迁移；必须使用原来源显式创建新任务。当前页面只支持人工核查，不写入校正文本；预留的 nullable 校正层不会覆盖原始 OCR。

HTML 审阅报告以一条命中为一张卡，严格按照“`#序号`、文件名、任务内相对路径和页码 → 高亮命中词的完整 OCR 上下文、置信度、人工结论和备注 → 完整原始页面”展示，页面图只标记当前命中。左侧 320px 目录覆盖全部当前筛选结果，显示序号、文件名、页码、命中词和最多两行摘要；完整相对路径可通过提示查看。点击目录项会自动切换分页并定位、短暂高亮相应卡片。目录在桌面端可折叠，窄窗口变为顶部可展开目录，打印时隐藏。报告支持文件和校对状态筛选、文字搜索、序号升序等排序、每页 10/20/50/100 条、大图按序号前后切换、返回顶部以及 A4 纵向打印。打印始终输出完整报告且每条命中独立分页，不受当前筛选和分页影响。报告不包含源文件绝对路径，不依赖 ArchiveLens 正在运行，也不加载外部资源。

检索词仅移除首尾 ASCII SPACE（U+0020），随后执行 NFC 规范化。匹配区分大小写，支持同一 OCR 行内的重叠精确字面匹配；不支持正则、通配符或跨 OCR 行匹配。任务创建后不能修改检索词。

## 暂停与恢复

运行中任务可暂停并退出。再次启动后，A11 任务从 SQLite 持久化 checkpoint 恢复，已正式提交的页面不会重新扫描。

Alpha10 已完成任务继续显示历史“约/約”检索条件。缺少可信页进度的旧未完成任务会显示 `LEGACY_TASK_REQUIRES_REVIEW`：旧结果保持只读，不会自动恢复或静默重扫；可使用任务页按钮把原目录带入新任务页面并重新扫描。

## 任务、诊断与本地数据

- “任务中心”可按任务名称、来源目录或检索词搜索，并按状态筛选和分页；从任一任务进入详情、校对或导出后，路由会保持该任务上下文。全局菜单首次启动默认以约 220px 展开，收起后为约 64px 图标栏；品牌和导航图标继续可用并通过提示显示名称，展开状态保存在本机。菜单不再显示“当前任务”卡片或未完成任务数量提示。
- 左侧底部的“设置”页面集中管理应用配置。“校对显示与扫描上下文”提供 6 种预设高亮颜色、自定义颜色、10%～60% 透明度、4 种横排/竖排阅读方向，以及关键词每侧 1～50 字的上下文长度；“简繁字形检索”提供简体、繁体或双向命中的全局默认。高亮和上下文配置支持全局默认和指定任务覆盖；出处页始终按源文件无损显示。旧任务会按扫描时 SHA-256 按页升级，无可信指纹或原文件内容已变化时会要求重新扫描。原始文件不会被修改。
- 若任务已经创建但启动请求失败，任务会保留为“待启动”，详情页提供“启动任务”重试；无需返回新建页重复创建。
- “环境诊断”保留为独立页面，不在左侧主菜单展示；从“设置”页面点击“打开环境诊断”进入。诊断将 PASS/WARN/FAIL 转换为“可用/受限/不可用”，逐项显示实际影响、处理建议，并提供重新检查和打开日志目录入口。
- 仅缺少 Tesseract 二次复核、DJVU 工具或中文语言包时，整体环境显示“部分能力受限”；主识别引擎、ONNX Runtime、工作目录或本地识别服务不可用时，才显示“不可用”。
- 任务页可打开任务目录和应用日志。导出中心展示数据库中持久化的最近导出作业和成功记录，重启应用后仍可查看；上次实际运行中的作业会标记为“已中断”，可重新导出。
- 任务中心可勾选当前页并顺序批量暂停、取消或删除，一次最多 20 条；翻页、搜索或改变筛选会清空选择。批量栏会先显示可执行和将跳过的数量，每项执行前重新核验状态，单项失败不阻塞后续任务。批量取消不再额外确认；批量删除会显示清单式二次确认并明确来源文件不会删除。结束后可查看成功、跳过和失败清单，并在重新核验状态后重试失败项；`cleanup_failed` 会作为重试清理处理。删除只清理 ArchiveLens 生成的任务记录、页面、校对和导出数据，永远不会删除来源文件。若文件占用或权限问题导致清理失败，任务仍会显示“清理失败”，可重试清理或打开残留目录；不要在资源管理器中手工删除正在运行的任务目录。
- 设置页“本地数据与隐私”显示数据库、任务派生文件、成功导出、导出临时残留、日志与设置占用，并可展开查看逐任务占用。统计不跟随链接；无法读取时会明确标记不完整。
- “清理安全临时残留”需要二次确认，只重试数据库登记且已结束的导出作业残留；不会删除数据库、OCR、校对、成功导出、活动作业、未知孤立目录或来源文件。
- 本地处理不等于加密保存：数据库、OCR 原文/索引、页面图片、校对备注和导出默认是本地明文。安装版与 Portable 默认共用 `%APPDATA%\ArchiveLens`，卸载默认保留这些数据。详见[本地数据、隐私与清理边界](privacy-and-local-data.md)。

## Alpha 限制

- 仅支持 Windows 10/11 x64。
- 安装包未签名，SmartScreen 可能显示未知发布者。
- PDF 主链路使用随包 pypdfium2/PDFium 与 RapidOCR；TIFF、JPEG、PNG 使用 Pillow 解码。
- 多页 TIFF 的每个 Frame 作为一个页面；JPEG、PNG 作为单页文档。图片单页最多 2 亿像素、最长边 30000 像素，多页 TIFF 最多 5000 页；不支持动态 PNG/APNG。
- 完整安装包已包含 Tesseract 二次复核、四个简繁中文语言包和 DJVU 工具；安装后无需另外下载。若包内文件损坏或缺失，诊断页会显示实际包内路径和处理建议。
- 文档与 OCR 结果默认只保存在本机，不上传网络。
