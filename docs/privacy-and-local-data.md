# 本地数据、隐私与清理边界

ArchiveLens 是本地优先应用：正式运行时不上传原始档案、OCR 内容或遥测，也不加载
远程网页。**本地处理不等于加密保存**。当前版本没有应用级透明加密，数据库、OCR
原文、索引、页面图片、校对备注和导出默认以本地明文保存。

## 数据位置与内容

安装版与 Portable 默认使用同一 Windows Electron `userData`：

```text
%APPDATA%\ArchiveLens\
├─ settings.json                 显示、检索和任务覆盖设置
├─ logs\
│  ├─ app.log / app.log.1        Electron 主进程日志
│  └─ engine.log / engine.log.1  Python Engine 诊断日志
└─ engine\
   ├─ archivelens.db             SQLite 任务、OCR 原文、索引、命中、校对和导出记录
   ├─ archivelens.db-wal/-shm    SQLite WAL 运行文件（存在时属于数据库的一部分）
   ├─ backups\                   schema 迁移前数据库备份及校验元数据（最近 3 对）
   ├─ tasks\<task_id>\
   │  ├─ scan\...                ArchiveLens 生成的页面图片、crop 和扫描派生文件
   │  └─ exports\...             已成功生成的 HTML/JSON/校对导出
   └─ .export-jobs\<export_id>\  导出作业专属临时目录
```

测试和受控开发运行可通过 `ARCHIVELENS_USER_DATA_DIR` 改到隔离目录；正式用户不需要
设置该变量。原始 PDF、DjVu、TIFF、JPEG 和 PNG 保留在用户选择的位置，ArchiveLens
保存只读引用及校验信息，不把原文件归类为缓存，也不会在任务删除或临时清理时删除。

设置页的“本地数据与隐私”会统计当前进程可读取的文件字节数，并分别显示数据库、迁移备份、
任务派生数据、成功导出、导出临时残留、日志和设置。统计不跟随符号链接或 Windows
junction；若存在无法读取的条目或链接，页面会明确标记“统计不完整”，不会把可读
小计描述成完整占用。

## 保留与删除合同

| 动作 | 会删除 | 不会删除 |
| --- | --- | --- |
| 取消/失败导出 | 该 `export_id` 的临时和未登记输出；失败时保留可重试诊断 | 其他导出、成功导出、任务数据、来源文件 |
| 设置页“清理安全临时残留” | 数据库已登记、已结束导出作业的专属临时/未登记输出 | 活动作业、未知孤立目录、数据库、OCR、校对、成功导出、来源文件 |
| 删除任务 | 该任务的数据库记录和 ArchiveLens 生成的任务目录 | 原始来源文件、其他任务 |
| 卸载安装版 | 应用程序文件 | 默认保留整个 `userData` |
| 关闭 Portable | 不自动删除数据 | 默认仍保留与安装版共用的 `userData` |

数据库迁移备份含 OCR、校对、任务和导出记录，敏感级别与主数据库相同，不是可随意删除的
缓存。Engine 只在真实 schema 升级前创建，默认保留最近 3 对；临时清理和任务删除都不会
处理它。用户若手工处置备份，必须先确认不再需要升级失败恢复或降级恢复。

任务删除和导出清理均使用数据库登记的标识与受信目录推导，拒绝 renderer 提供任意
绝对路径。清理失败会保留状态并允许重试。设置页的临时清理有二次确认；未知孤立目录
不会被自动猜测删除，避免把用户数据或其他程序目录误当缓存。

删除任务或文件只表示文件系统层面删除，**不等于物理介质安全擦除**。同步盘版本、
系统备份、用户复制的导出和其他备份介质不会被 ArchiveLens 一并删除。需要销毁敏感
数据时，应先确认所有副本和组织的数据处置要求，再使用操作系统或存储设备提供的合规
流程；ArchiveLens 当前不提供安全擦除保证。

## 威胁模型与用户责任

- 同一 Windows 账户下具有文件读取权限的其他程序，可能读取 ArchiveLens 明文数据。
- 账户被入侵、恶意软件、未加密磁盘、云同步或系统备份都可能扩大数据暴露范围。
- 日志设计为不记录 OCR 全文或密钥，但可能包含任务标识、组件路径和失败诊断；提交日志
  给第三方前仍应检查并按敏感资料处理。
- HTML/JSON 导出可能包含 OCR 全文、命中上下文、页面图片、备注和人工结论；导出后由
  用户决定保存位置，ArchiveLens 无法控制后续复制或分享。
- 建议使用独立且受保护的 Windows 账户、系统提供且适用的磁盘加密、最小文件权限和受控
  备份；不要把敏感导出放入未获授权的同步目录。

ArchiveLens 不承诺防御已经控制当前 Windows 账户或具有管理员权限的攻击者。当前边界的
目标是无网络外发、最小 IPC 权限、安全的应用内清理和可解释的数据保留行为。

## 备份、升级与降级

当前稳定候选会在数据库 schema 迁移前建立通过完整性和 SHA-256 校验的备份，并在迁移
失败时自动恢复。备份同样是本地明文，卸载默认保留。实现、保留数量、历史版本限制和
安全降级步骤见 [`migration.md`](migration.md)。

手工备份时必须在 ArchiveLens 完全退出后复制 `engine/archivelens.db` 及同时存在的
`-wal`/`-shm`，或使用项目提供并已验证的备份流程。不要只复制正在 WAL 模式写入的主
数据库文件。降级时旧程序不得直接写入未来 schema；应在应用完全退出时安装旧版本并
恢复升级前备份。历史 `alpha.10` 已实证会把未来 schema 改写为 v2，因此不得用它试开
当前数据。

## 查看与诊断

- 设置 → 本地数据与隐私：查看总占用、逐任务占用、打开 `userData` 和重试
  安全临时清理。
- 任务详情：打开该任务受控目录。
- 导出中心：打开受控导出目录，查看作业状态、清理错误并重新导出。
- 开发者模式（隐藏入口）：查看 Engine、原生组件与本地工作目录，打开日志目录与本地数据目录。日志入口只在开发者模式下可用，普通页面与系统托盘不再提供。

ArchiveLens 不会自动按时间删除任务历史。是否保留或删除正式任务属于用户决定，不能被
“临时文件清理”替代。

## 开发者模式与诊断信息

开发者模式默认关闭，且不是账户或权限系统——它假定应用由当前 Windows 账户所有者本地使用。开启后可复制三种诊断信息，全部只写入本机剪贴板、绝不自动上传：

- 脱敏诊断摘要：仅版本、状态、错误码、任务 ID、检查结果与路径是否存在；移除用户名、完整路径、文件名、OCR 内容与日志。
- 含完整路径诊断摘要：保留完整路径、文件名与原始错误，但不含 OCR 上下文与日志。
- 完整 AI 错误调试信息：不脱敏，包含完整快照、用户名、路径、文件名、原始错误、调用栈、当前 OCR 上下文及最近 300 行日志。请仅提供给你信任的调试对象。

诊断信息绝不枚举或复制环境变量、令牌、Cookie、密钥或其他凭据。
