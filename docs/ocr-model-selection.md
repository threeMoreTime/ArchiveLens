# 统一简繁 OCR 模型选择记录

本记录保存 ArchiveLens 统一文字识别模型的可复核工程结论，不包含用于本地评测的
用户 PDF、页面图片、裁剪图或 OCR 文本。

## 决策与门禁

候选模型均由同一 `rapidocr-onnxruntime==1.4.4` 推理运行时加载，只替换文字识别
ONNX；检测与方向分类继续使用既有模型。批准的选择门禁为：

- 真实档案样本准确性更高；
- 相对候选模型的推理耗时增加不超过 30%；
- 额外峰值工作集不超过 512 MB；
- 必须能保留并区分简体、繁体和主流地区字形；
- 最终安装包只包含一个文字识别模型；
- 全部能力本地运行、零付费、无遥测和用户数据外发。

## 本地真实档案基准

2026-07-16 在 Windows x64、Python 3.11.9 上，从一份 993 页本地历史档案 PDF
等距抽取 36 页。原文件、渲染页面、分歧截图和完整识别结果只保存在忽略目录，
不得进入 Git 或发布产物。

| 指标 | PP-OCRv5 mobile | PP-OCRv6 small |
| --- | ---: | ---: |
| 36 页识别总耗时 | 276.781 s | 181.190 s |
| 单页中位耗时 | 7.573 s | 4.888 s |
| 识别文本行 | 1,146 | 1,231 |
| 识别字符 | 18,099 | 19,953 |
| 独立进程峰值工作集 | 207.4 MB | 210.3 MB |

两模型共有 1,146 个可对齐文本框，其中 83 行文本完全相同；PP-OCRv6 额外检出
85 行。对跨卷首、正文和后段的分歧页面进行人工对照后，PP-OCRv6 在历史竖排
文本、年号和常用繁体字上的整体可读性更高，因此通过准确性优先门禁。

## 字形复核与候选层

回归夹具暴露出 PP-OCRv6 对整行 `约 約` 的上下文结果会混淆第一个字形，但同一
模型根据字符框分别识别时，可高置信区分两个字形。最终适配层采用以下保守规则：

1. 整行上下文识别结果原样保留；
2. 只有上下文字符置信度不高于 0.85 时才触发字符框复核；
3. 字符框置信度至少为 0.97，且比上下文结果至少高 0.15；
4. OpenCC 必须证明两个字符属于同一简繁字形族；
5. 词义不同、字形族不同的候选不得替换；
6. 孤立单字不自动替换，保留同一模型的 Top-5 CTC 候选供搜索和人工复核。

该适配层在 6 个同页、同进程成对样本上的总耗时相对纯 PP-OCRv6 增加 12.68%；
独立进程峰值工作集由 218.5 MB 增至 254.1 MB，增加 35.6 MB，均在批准门禁内。

## 最终选择

最终锁定 `PP-OCRv6-small`。运行时对模型文件执行 SHA-256 校验；构建脚本移除
RapidOCR wheel 自带且未使用的 PP-OCRv4 文字识别模型，并验证包内只有一个
`PP-OCRv6_rec_small.onnx`。模型公开再分发权利仍由公开发布前人工合规门禁控制，
本技术选择不构成正式发布授权或法律意见。
